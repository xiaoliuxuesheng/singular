# 一 Session跨域

- 跨域

  > 客户端请求的时候,请求的服务器不是同一IP, 端口, 域名, 主机名的时候就称为跨域

- 域

  > 在应用模型, 一个完整的, 有独立访问路径的功能集合称为一个域	
  >
  > 域的划分 : IP 域名 端口 主机名为标准

- Session跨域是摒弃了系统提供的Session, 而是使用自定义的类似Session的机制来保存客户端数据的解决方案

  > 使用cookie传值, cookie中保存session_id作为key, 客户端数据作为value,实现session跨域
