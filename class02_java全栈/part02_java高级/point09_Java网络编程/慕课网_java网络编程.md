2-1 URL解析

1. URL标准结构

   ```url
   http://www.google.com:80/search?query=test
   ```

   - 协议：`http://`
   - 域名/IP：`www.google.com`
   - 端口：`80`
   - 资源路径：`search`
   - 条件：`?query=test`

2-2 DNS解析

1. DNS解析：计算机之间是通过IP定位的，但是IP很那记忆，所以需要定义一组域名对应一个IP地址，如果从域名解析为IP称为域名解析，

   - 域名的完整结构：www.google.com.root；域名解析是从右向左

     | 域名            | 说明                         |
     | --------------- | ---------------------------- |
     | .root           | 根域名。所以域名都是。root   |
     | .com、.edu、org | 顶级域名                     |
     | google          | 次级域名：是值公司标识       |
     | www             | 主机名：发布网站时候的自定义 |

   - DNS域名解析就是根据域名获取到注册到网络时候的主机IP

2-3 网络协议

- 网络协议
  - 应用层：用户接触到应用可以看到的数据：协议http ftp smtp
  - 传输层：端口对端口的数据的传输所规定的规格 tcp udp
  - 网络层：数据在网络中传输的规则 ip
  - 链路层：数据在网卡交互的规则 Ethernet 以太网
  - 实体层：网络机器之间的物流链接，网线或无线 电信号协议
- 计算机之间的物流链接

2-7 链路层数据包格式

- 链路层
  - 帧：1518字节
    - Ethernet头：帧开始的18个字节，包含网卡的Mac地址
    - Ethernet数据：传输的数据，1帧最多的数据1500个字节
- 网络层
  - 帧
    - Ethernet头
    - Ethernet数据
      - IP表头
      - IP数据
- 传输层
  - IP数据
    - TCP表头
    - TCP数据
