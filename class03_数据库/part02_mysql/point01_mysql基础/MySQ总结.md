# 第一章 初识MySQL

## 1.1 数据库术语

- 数据库：
- 表：关系型数据库中，数据库表是一系列二维数组的集合，存储数据和操作数据的逻辑结构；
- 行：是数据的组织单位，一行表示一组数据的集合
- 列：被成为字段，每一列表示记录的一个属性，每一列都有相同的基本属性：数据类型、长度
- 数据类型：数据类型决定数据在计算机中的存储格式，常用的数据类型：整数数据类型、浮点数数据类型、精确小数类型、二进制数据类型、日期数据类型、字符串数据类型；
- 主键：用于唯一标识表中的每一条记录，主键列的特点非空且唯一，每个表只定义一个主键，一个主键可以定义多列

## 1.2 数据库技术

- 数据库的构成：硬件部分（存储数据库中的数据，指计算机、存储设备等待）、软件部分数据的操作系统
- 数据库系统：3个组成部分
  1. 数据库：存储数据的地方
  2. 数据库管理系统：管理数据库的软件
  3. 数据库应用程序：管理数据库的软件的补充，提高数据库的处理能力
- SQL语言：结构化查询语言；对数据库进行查询和修改操作
  1. 数据定义语言（DDL）：DROP、CREATE、ALTER
  2. 数据操作语言（DML）：INSERT、UPDATE、DELETE
  3. 数据查询语言（DQL）：SELECT
  4. 数据控制语言（DCL）：GEANT、REVOKE、COMMIT、ROLLBACK
- 数据库接口访问：
  1. ODBC
  2. JDBC
  3. ADO.NET
  4. PDO

## 3.1 数据库的操作

### 1. 数据库介绍

- **数据库引擎**：不同的存储引擎提供不同的存储机制、索引技巧、锁定水平等功能

  | InnoDB     | 支持ACID，简单地说就是支持事务完整性、一致性；<br />支持行锁，以及类似ORACLE的一致性读，多用户并发；<br />独有的聚集索引主键设计方式，可大幅提升并发读写性能；<br />支持外键；<br />支持崩溃数据自修复；<br />InnoDB设计目标是处理大容量数据库系统,CPU利用率极高<br />它是一个可靠地事务处理引擎，不支持全文本搜索 |
  | ---------- | ------------------------------------------------------------ |
  | **MyISAM** | **不支持事务每次查询具有原子性<br />只支持表锁<br />强调的是性能，其执行速度比InnoDB类型更快，但是不提供事务支持<br />如果执行大量的SELECT（删除任务），MyISAM是更好的选择** |
