# 05_Typescript装饰器第七章 模块

## 7.1 模块的介绍

- Typescript中的模块分为 **内部模块** 和 **外部模块** : 一个Typescript的ts脚本文件为一个模块
    - 内部模块 : 指的是当前脚本文件中的所有的成员
    - 外部模块 : 当前脚本文件之外的其他的文件
- 模块的特点 : 模块之间的作用域相互独立,模块在各自的作用区域中执行
- Typescript脚本之间如果需要外部脚本使用,需要**导出** 和 **导入** 功能进行实现
    - 关键字 export 和 import 完成

## 7.2 模块导入和导出

### 1. 导出 & 导入一个成员

```js
// 导出
	export 模块成员名称; // 变量 方法 命令空间 类
// 导入
	import {模块成员名称} from "模块所在文件的路径";
```

### 2. 导出 & 导入多个成员

```js
// 导出
	export 模块成员1名称; // 变量 方法 命令空间 类
	export 模块成员2名称; // 变量 方法 命令空间 类
// 导入
	import {模块成员1名称,模块成员2名称} from "模块所在文件的路径";
```

### 3. 默认导出

- 默认导出表示一个模块只可以有一个导出

    ```js
    // 导出
    	export default 模块成员名称; // 变量 方法 命令空间 类
    // 导入
    	import 模块成员名称 from "模块所在文件的路径";
    ```

    > 导入时候不需要大括号

### 4. 设置导入别名

```js
	import {模块成员1名称,模块成员2名称 as 别名} from "模块所在文件的路径";
```

## 7.3 命令空间

### 1. 命令空间特点

- 命令空间用于每部模块 : namespace
- 命名空间之间的作用域是相互独立 : 主要用于组织代码,避免命名冲突
- 同一个内部模块可以有多个命名空间

### 2.命名空间的定义 导出和导入

- 定义命名空间

    ```js
    namespace 命名空间名称 = {
        成员变量
        成员类
        成员方法
    }
    ```

- 导出命名空间

    ```js
    export namespace 命名空间名称= {
        成员变量
        成员类
        成员方法
    }
    ```

- 命令空间的导入与使用

    ```js
    // 导入
    	import {命名空间名称} from "命令空间所在脚本的路径";
    // 使用命名空间的成员 : 点语法
    	命名空间名称.成员变量
    	命名空间名称.成员方法
    	... ...
    ```
